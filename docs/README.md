## 🚀 기능 목록

### 로직 분리에 따른 예외 발생 기준

- 예외 발생 시 적합한 표준예외를 사용한다.
- 입력값 오류와 코딩 오류를 구분하기 위해 에러메시지에 `[ERROR] 게임 기능 오류:`, `[ERROR] 사용자 입력 오류:`와 같이 표기한다.

### 도메인 구성 요소
- 지하철역
  - 이름
- 지하철 노선
  - 이름
- 노선 경로 (구간의 모음)
    - 노선 이름
    - 해당 노선의 상행 종점 ~ 하행 종점까지의 모든 구간의 역 정보 (순서대로)
- 지하철역 목록
- 지하철 노선 목록
- 노선 경로 목록

### 도메인 로직을 위한 기능

### 역 관리
1. - [ ] 지하철 역 목록 조회
2. - [ ] 지하철 역 등록
   - 역 이름을 전달받아 지하철 역 목록에 해당 역을 등록한다.
   - [ ] `예외발생` : 중복된 지하철 역 이름 
   - [ ] `예외발생` : 조건에 맞지 않는 지하철 역 이름(2글자 이상) 
3. - [ ] 지하철 역 삭제
   - 역 이름을 전달받아 지하철 역 목록에서 해당하는 이름의 역을 삭제한다.
   - [ ] `예외발생` : 노선에 등록된 역

### 노선 관리
1. - [ ] 지하철 노선 목록 조회
2. - [ ] 지하철 노선 등록
   - 전달받은 노선 이름으로 만든 새 노선 정보를 지하철 노선 목록에 등록한다.
   - 등록할 때 노선 이름, 상행 종점역, 하행 종점역으로 이루어진 구간 정보를 생성해 노선 경로 목록에 저장한다.
   - [ ] `예외발생` : 중복된 지하철 노선 이름
   - [ ] `예외발생` : 존재하지 않는 종점역 이름
3. - [ ] 지하철 노선 삭제
   - 노선 이름을 전달받아 지하철 노선 목록에서 해당하는 이름의 노선을 삭제한다.
   - 노선 이름을 전달받아 노선 별 구간 목록에서 해당하는 노선의 구간 정보를 삭제한다.

### 구간 관리
1. - [ ] 지하철 노선도 조회
   - 노선 경로 목록에 저장된 정보를 이용해 각각 상행 종점부터 하행 종점까지 연결된 순서대로 역 목록을 조회한다.
2. - [ ] 구간 추가
   - 노선 경로 목록에서 해당 노선 경로를 찾아 전달받은 역을 전달받은 순서 위치에 삽입한다. (뒤에 있는 역들은 하나씩 뒤로 이동)
   - [ ] `예외발생` : 존재하지 않는 노선 이름
3. - [ ] 구간 삭제
   - 노선 경로 목록에서 해당 노선 경로를 찾아 전달받은 이름을 가진 역을 삭제한다. (뒤에 있는 역들은 하나씩 앞으로 이동)
   - [ ] `예외발생` : 존재하지 않는 노선 이름
   - [ ] `예외발생` : 존재하지 않는 역 이름
4. - [ ] 프로그램 종료
5. - [ ] 사용자의 잘못된 입력으로 인한 예외 발생 시 해당 기능의 관리 화면으로 돌아가는 기능

### UI 로직을 위한 기능

1. - [ ] 메인 화면 출력
2. - [ ] 기능 번호 입력 (각 화면에서 사용)
3. - [ ] 역 관리 화면 출력
4. - [ ] 노선 관리 화면 출력
5. - [ ] 구간 관리 화면 출력
6. - [ ] 각 세부 기능 화면에서 메인 화면으로 돌아가는 기능
7. - [ ] 정보 및 결과 메시지 출력 
   - 역 목록
   - 노선 목록
   - 지하철 노선도
   - 특정 기능 완료 메시지 
8. - [ ] 안내 메시지 출력
9. - [ ] 에러 메시지 출력
